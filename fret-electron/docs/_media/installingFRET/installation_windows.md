    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-installing-fret-on-windows-10" class="anchor" aria-hidden="true" href="#installing-fret-on-windows-10"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installing FRET on Windows 10</h1>
<p>These installation instructions describe the installation of FRET
natively under Windows 10.</p>
<h2><a id="user-content-prerequisites" class="anchor" aria-hidden="true" href="#prerequisites"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Prerequisites</h2>
<p>The following software packages must be installed.</p>
<ul>
<li>
<p>install git.
Git is available from <a href="https://git-scm.com/download/win" rel="nofollow">https://git-scm.com/download/win</a></p>
</li>
<li>
<p>install python 2.7 from <a href="https://www.python.org/downloads/release/python-2718/" rel="nofollow">https://www.python.org/downloads/release/python-2718/</a></p>
<ul>
<li>Note: The installation of npm requires that python2.7 is installed as a windows package. A python installation under cygwin does not seem to be working</li>
</ul>
</li>
<li>
<p>install the node version manager. This step allows the user to have more than one version of nodejs running on the machine. This feature might be helpful, since FRET requires certain versions of nodejs.
To do so, follow the Microsoft tutorial at <a href="https://docs.microsoft.com/en-us/windows/nodejs/setup-on-windows" rel="nofollow">https://docs.microsoft.com/en-us/windows/nodejs/setup-on-windows</a></p>
</li>
<li>
<p>install the nvm-setup from <a href="https://github.com/coreybutler/nvm-windows/releases">https://github.com/coreybutler/nvm-windows/releases</a>. Download the repo as a .zip file, unpack, and install</p>
</li>
</ul>
<h2><a id="user-content-test-the-nvm-installation" class="anchor" aria-hidden="true" href="#test-the-nvm-installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Test the nvm Installation</h2>
<p>In a powershell, issue <code>nvm ls</code>. Unless you already have a version of nodejs installed under windows, no output should be produced.</p>
<p>Type <code>nvm list available</code> to show, which nodejs versions are available.
For FRET, we suggest to install 10.15.0.
This can be accomplished by <code>nvm install 10.15.0</code>.
After the installation, the <code>nvm ls</code> should list the installed version:</p>
<p><a target="_blank" rel="noopener noreferrer" href="/NASA-SW-VnV/fret/blob/master/fret-electron/docs/_media/screen_shots/nvm_installed_nodejs.png"><img src="/NASA-SW-VnV/fret/raw/master/fret-electron/docs/_media/screen_shots/nvm_installed_nodejs.png" style="max-width:100%;"></a></p>
<p>Make sure that you select the version 10.15.X as the currently active one.</p>
<h2><a id="user-content-download-fret" class="anchor" aria-hidden="true" href="#download-fret"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Download FRET</h2>
<p>Make a directory where the FRET sources should reside and clone the
FRET repository</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="mkdir FRET
cd FRET
git clone https://github.com/NASA-SW-VnV/fret.git

cd FRET

cd .\fret\fret-electron

<h2><a id="user-content-install-the-node-windows-build-tools" class="anchor" aria-hidden="true" href="#install-the-node-windows-build-tools"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Install the node-windows-build tools</h2>
<p>On an adminitrator powershell execute</p>
<p><code>npm install --global windows-build-tools</code></p>
<h2><a id="user-content-install-cygwin" class="anchor" aria-hidden="true" href="#install-cygwin"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Install cygwin</h2>
<p>A cygwin installation is currently needed to provide the <code>gcc</code> C compiler.</p>
<p>From (<a href="https://cygwin.com/install.html" rel="nofollow">https://cygwin.com/install.html</a>) download the `setup-x86_64.exe'.
Execute this binary.
In the GUI select the default selection and manually add the following
packages</p>
<ul>
<li>make
*gcc-core</li>
</ul>
<p>These packages can be best selected using the search function in the selection window. Then continue the installation.
This installation should produce an icon for cygwin terminal on the Desktop.</p>
<h2><a id="user-content-test-for-make-and-gcc" class="anchor" aria-hidden="true" href="#test-for-make-and-gcc"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Test for make and gcc</h2>
<p>Start the cygwin terminal and type <code>make</code>.
It should respond with an error like
<code>No targets specified and no makefile found</code>.
If <code>command not found</code> is shown, make has not been installed properly.</p>
<p>Similarly, type <code>gcc</code> in the terminal.
It should respond with <code>gcc: fatal error: no input files</code>.
If <code>command not found</code> is shown, the C compiler has not been installed properly.</p>
<p><a target="_blank" rel="noopener noreferrer" href="/NASA-SW-VnV/fret/blob/master/fret-electron/docs/_media/screen_shots/cygwin_test_prerequisites.png"><img src="/NASA-SW-VnV/fret/raw/master/fret-electron/docs/_media/screen_shots/cygwin_test_prerequisites.png" style="max-width:100%;"></a></p>
<h2><a id="user-content-install-fret" class="anchor" aria-hidden="true" href="#install-fret"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Install FRET</h2>
<p>In the cygwin terminal, go into the FRET directory. Since the FRET
directory is under the Windows Users directory, the command is
<code>cd /cygdrive/c/Users/USERNAME/fret/fret-electron</code></p>
<p>Type <code>npm run fret-install</code> for a fresh installation of FRET or
<code>npm run fret-reinstall</code> if there is already an existing (working or
failed) installation of FRET</p>
<p>Finally, type <code>npm start</code> to launch FRET</p>
<p><a target="_blank" rel="noopener noreferrer" href="/NASA-SW-VnV/fret/blob/master/fret-electron/docs/_media/screen_shots/fret-on-windows.png"><img src="/NASA-SW-VnV/fret/raw/master/fret-electron/docs/_media/screen_shots/fret-on-windows.png" style="max-width:100%;"></a></p>
<p>Notes:</p>
<ul>
<li>if FRET fails to start up after a seemingly successful installation,
do a <code>npm run fret-reinstall</code>.</li>
